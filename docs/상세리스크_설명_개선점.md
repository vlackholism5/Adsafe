# 상세 리스크 항목 설명 — 사용자 관점 개선점

## 1. 판정 정확도 (버그 수정)

| 문제 | 개선 내용 |
|------|-----------|
| **40% 할인을 "50% 이상"으로 잘못 표시** | 50% **초과**만 고위험으로 판정하도록 수정. 40% 할인은 "50% 이상" 항목으로 잡히지 않음. (룰: `\d+% 할인` 패턴 제거, 이벤트 로직에서 `pct > 50`만 적용) |

---

## 2. 설명 문구 — 사용자 혼란 방지

| 항목 | 현재 문제 | 개선 방향 |
|------|-----------|------------|
| **할인/이벤트 (금전 혜택)** | "기간·조건을 명시하지 않으면 제한될 수 있습니다" → 사용자가 **기간(2026.01.01~01.31)을 이미 적었는데도** 같은 문구가 나와 "뭘 더 써야 하지?"라고 느낄 수 있음 | "이벤트 기간, 할인율, 적용 대상을 **함께 명시하면** 리스크를 줄일 수 있습니다"처럼 **안내형**으로 변경. 이미 기간을 쓴 경우에도 "추가로 할인율·적용 대상 명시를 권장"이라는 맥락이 드러나도록 함 |
| **50% 초과 할인** | "50% **이상**"이라는 표현이 50% 포함 여부를 헷갈리게 함 | 카테고리/설명을 "**50% 초과**"로 통일해, 50%는 해당하지 않고 51% 이상만 고위험임을 분명히 함 |

---

## 3. 중복·과다 표시

| 문제 | 개선 방향 |
|------|------------|
| **"할인"과 "이벤트"를 각각 별도 항목으로 표시** | 같은 문장("40% 할인 이벤트")에서 나온 표현인데 2건으로 나뉘어, 사용자가 "같은 말을 두 번 지적당한 느낌"을 받을 수 있음 | 가능하면 **하나의 리스크로 묶어** "할인·이벤트 표현"처럼 표시하거나, 한 건만 노출하고 설명에서 "할인·이벤트 등"으로 포괄 안내 |
| **기간을 이미 썼는데 "이벤트 기간 미표기"** | 2026.01.01~01.31처럼 기간이 있음에도 "기간 미표기"로 나오면 신뢰가 떨어짐 | 기간이 인식되면 "이벤트 기간 미표기" 판정을 하지 않도록 로직 보완 (이미 반영된 경우, 설명 문구만 위 2번처럼 안내형으로 정리) |

---

## 4. 수정 가이드 — 실행 가능하게

| 항목 | 개선 방향 |
|------|------------|
| **"이벤트 기간, 할인율, 참여 조건을 명시하세요"** | 조금 추상적일 수 있음 | "이벤트 기간(시작일~종료일), 할인율(예: 40%), 적용 대상(예: 턱보톡스 1회)"처럼 **예시**를 넣으면 사용자가 바로 고쳐 넣기 쉬움 |
| **"할인율을 50% 미만으로"** | 50%는 되는지 안 되는지 애매할 수 있음 | "50% **이하**" 또는 "51% 이상 할인은 고위험"처럼 **기준을 한 번 더 명시** |

---

## 5. 적용한 코드 반영 사항

- **RISK_PRICE_EXCESSIVE**: `\d+% 할인` regex 제거 → 50% **초과**일 때만 이벤트 로직에서 고위험 추가. level3 문구 "50% 이상" → "50% 초과"로 변경.
- **RISK_INDUCEMENT_DISCOUNT**: 설명을 "기간·조건 미명시 시 제한" → "기간·할인율·적용 대상을 함께 명시하면 리스크 감소" 안내형으로 변경.

위 개선점은 사용자가 **"왜 이게 리스크인지"**, **"뭘 고치면 되는지"**를 더 쉽게 이해하도록 하기 위한 목록입니다.
